# ИС учёта операционных инцидентов в отделах продаж

Дипломная работа: информационная система для регистрации, обработки и аналитики операционных инцидентов в отделах продаж.

## Онлайн-демо

Развёрнутая версия доступна по ссылке:

**https://serebrennikov-incidents-mvp-production.up.railway.app**

Вход в систему:
- **E-mail:** `admin@example.com`
- **Пароль:** `admin12345`

Замечания:
- Первое открытие после периода простоя может занять 10–20 секунд («холодный старт» контейнера).
- Файлы, которые вы прикрепляете к инцидентам, хранятся во временной файловой системе контейнера и могут быть удалены при следующем перезапуске сервиса. Все остальные данные (инциденты, пользователи, справочники, аудит) сохраняются в БД и переживают перезапуски.

Если хочется развернуть систему у себя, инструкция ниже.

## Стек

- **Backend:** Python 3.12, FastAPI, SQLAlchemy 2.0, PostgreSQL 16
- **Frontend:** React 18, TypeScript, Vite, Ant Design 5
- **Запуск:** Docker Compose

## Запуск на своём компьютере

Прототип запускается одной командой через Docker. Устанавливать Python, Node.js или PostgreSQL отдельно не требуется — все сервисы поднимаются внутри контейнеров автоматически.

### Что нужно установить заранее

1. **Docker Desktop** — программа, которая запустит все сервисы системы (базу данных, сервер, веб-интерфейс) в изолированных контейнерах.
   - **Windows / macOS:** [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop) → скачать установщик → запустить → перезагрузить компьютер по запросу. После установки откройте Docker Desktop и дождитесь, пока в трее (Windows) или в строке меню (macOS) появится статус «Docker Desktop is running».
   - **Linux:** следуйте инструкции на [docs.docker.com/engine/install](https://docs.docker.com/engine/install/) для вашего дистрибутива.

2. **Git** — для скачивания исходников: [git-scm.com/downloads](https://git-scm.com/downloads). Если устанавливать Git не хочется — можно скачать ZIP-архив проекта с этой страницы (зелёная кнопка `Code` → `Download ZIP`) и распаковать в любую папку.

### Запуск

1. **Скачайте проект** одним из двух способов:
   - Через Git:
     ```
     git clone https://github.com/serebre2nikovdanil/Serebrennikov-incidents-mvp.git
     cd Serebrennikov-incidents-mvp
     ```
   - Через ZIP: распакуйте архив, откройте получившуюся папку в терминале (PowerShell на Windows, Terminal на macOS/Linux).

2. **Создайте файл с настройками** из шаблона. Менять значения внутри не нужно — для локального запуска подходят значения по умолчанию.
   - **Windows (PowerShell):**
     ```powershell
     Copy-Item .env.example .env
     ```
   - **macOS / Linux:**
     ```bash
     cp .env.example .env
     ```

3. **Запустите систему:**
   ```
   docker compose up --build
   ```
   Первый запуск займёт **5–15 минут** — Docker скачает образы PostgreSQL, Python и Node.js, установит зависимости и соберёт всё. Последующие запуски занимают несколько секунд. Команда не возвращает управление и продолжает выводить логи: пока окно терминала открыто, система работает.

4. **Дождитесь готовности.** В логах появятся сообщения:
   - `Application startup complete` — сервер готов;
   - `ready in NNN ms` — веб-интерфейс готов.

5. **Откройте в браузере:** http://localhost:5173

### Вход в систему

При первом запуске автоматически создаётся учётная запись администратора с правами на все разделы:

- **E-mail:** `admin@example.com`
- **Пароль:** `admin12345`

Под этим аккаунтом можно создать остальных пользователей системы (раздел «Пользователи»). Для пользователей доступны три роли: менеджер, руководитель и администратор.

### Демонстрационные данные (опционально)

В прототипе предусмотрен скрипт, который заполняет систему случайными инцидентами за последние 90 дней — чтобы оценить дашборд аналитики на реалистичных данных. Запуск (в новом окне терминала, пока система работает):

```
docker exec incident-backend python -m app.shared.demo_seed
```

### Остановка

Чтобы корректно остановить систему — нажмите `Ctrl+C` в окне с запущенным `docker compose`. Если этого недостаточно, в новом окне терминала выполните:
```
docker compose down
```

### Сброс (полная очистка с удалением данных)

Если хочется начать заново — например, удалить тестовые инциденты и снова получить чистую БД с одним администратором:
```
docker compose down -v
```
Флаг `-v` дополнительно удаляет хранилище базы данных. Следующий `docker compose up --build` создаст пустую систему.
